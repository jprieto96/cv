Portfolio


MySpace

Ongoing project. It helps the user to manage their own notes, actions, news. I am building a simple API with Java and Spring Boot connected to SQL to store the necessary data. I plan to migrate to GraphQL to practice with graph and binary tree based databases. The frontend is going to be a reactive web where the user will be able to choose his favorite news web pages, choose his actions and make a personal follow up of them. I have also thought about doing this project to practice my CI/CD techniques, I have made a pipeline with github actions and execute it in every push or pull request to the master branch. The pipeline consists of running all the tests and generating the JAR with maven. If the pipeline is called by a push to the master branch I also build an ubuntu based docker image to deploy the application inside, and once the docker image has been built, I upload the docker image to my DockerHub. Here you have my bakend repository of this project, where you can see more in detail the pipeline with github actions that I have commented before. github.com/jprieto96/myspace-api

Software Architecture - Design Patterns

Implement reusable and maintainable software over time by applying a good architecture from the beginning and optimal design patterns so that any change, migration, etc., is as inexpensive as possible. Another benefit of this, is the ease of testing the units of our code separately by mocking everything that comes out of the layer to be tested, all its dependencies. You can see in my GitHub the projects in which I apply many of these patterns, such as repository, factory, command, singleton, abstract factory, proxy, etc.
